I: Understanding item IDs



How they work:

Item IDs are numbers corresponding to an object or obstacle in a level.

Example:

0 = air

1 = floor

2 = jump pad

3 = glass

9 = stomper

You can find a list of all item IDs here or here.



Level Formation:

A Rolling Sky level is made up by a lot of item IDs on a text file.

Example:

(Demo made by sqdldev)



Loader Tiles:

Loader tiles divide a level into segments. Segments should be 7-15 lines long for a level to load. Tiles 181 through 188 are loader tiles.



II: Understanding geobuffers



What are they?:

Geobuffers are the way Rolling Sky knows how to render models and other effects. Geobuffers include how many and what type of object to render.

Example:

17,4,z = riser

18,4,z = floater

Here is a list


How they work:

Geobuffers work in a x,y,z format

x = the object/model and how its going to be displayed.

y = the type of geobuffer

z = maximum amount of the object in a segment

Here you can find a list of all geobuffers.

Note: Most geobuffers use geobuffer type 4 but some use other geobuffer types. You can check your geobuffer type here:

(Inspector made by sqdldev)


How to edit them:

1. Open UABE

2. Select file > open

3. Go to your Rolling Sky installation folder > Rolling Sky_Data

4. Select the resources.assets file and open it

5. Go to view > search by name

6. Search the following “geoBufferX” (X means the level id of the level geobuffer you want to edit. Here is a list)

7. Select plugins > export to .txt

8. Save the geobuffer file somewhere you can remember

9. Open Notepad ++ (dont close UABE)

10. Edit the geobuffer and save it (you can save the file by clicking Ctrl and S at the same time)

11. Go back to UABE and select plugin > import from .txt

12. Select the edited geobuffer

13. Save the resources.assets by clicking file > save

Note: UABE wont let you save the resources.assets in its original location while its open. You have to save it somewhere else and close UABE. After closing UABE you can move the resources.assets Rolling Sky installation > Rolling Sky_DATA

III: Custom routes



1. Open UABE

2. Select file > open

3. Go to your Rolling Sky installation folder > Rolling Sky_Data

4. Select the resources.assets file and open it

5. Go to view > search by name

6. Search the following “levelX” (X means the level id of the level you want to edit. Here is a list)

7. Select plugins > export to .txt

8. Save the level file somewhere you can remember

9. Open Notepad ++ (dont close UABE)

10. Edit the level and save it (you can use this level as an example for what you can and cannot do)

11. Go back to UABE and select plugin > import from .txt

12. Select the edited level

13. Save the resources.assets by clicking file > save

Note: Remember you have to add loader tiles every 7-15 lines

Note: UABE wont let you save the resources.assets in its original location while its open. You have to save it somewhere else and close UABE. After closing UABE you can move the resources.assets Rolling Sky installation > Rolling Sky_DATA


How to edit with Tiled:

1. Do all the steps above until step 8.

2. Open the file and under [header], look for the height of the level and remember it

3. Go to Tiled and create a new tilemap. The width for the map should be 5 and the height should be the same height as the original level. The tile size depends on what tileset you use

4. Import your tileset to the tilemap

5. Make your route. Once you finish editing save the tilemap

6. Open the tilemap in Notepad ++

7. Copy all the numbers from the tilemap to the level file (Only copy the numbers)

8. Save the level file somewhere you can remember

9. Continue with step 11


IV: Custom themes (Order of themes)

1. Open dnSpy

2. Select file > open

3. Go to your Rolling Sky installation folder > Rolling Sky_Data > Managed

4. Select the Assembly-CSharp.dll file and open it

5. Select the magnifying glass next to the start button to search

6. Search “LevelDesigner” and double click on the first option

7. Scroll to line 711. In this line you will find the code for all of the theme changes in rs. The code is in order of level IDs. It should look something like this:

new LevelDesigner.WorldTheme(new LevelDesigner.Theme[]
{
new LevelDesigner.Theme(0, 1, 1f),
new LevelDesigner.Theme(110, 2, 1f),
new LevelDesigner.Theme(212, 3, 1f, true),
new LevelDesigner.Theme(317, 6, 1f, false, false, false, false, false, false),
new LevelDesigner.Theme(420, 1, 1f)
}),

(Level 1 "Massif" theme changes)

8. Search for the level you want to change the theme. Right click the code and select "Edit Class (C#)" to edit it. Each row of numbers indicates a theme change. The first number indicates in what line of the level file the theme is going to change. The second number is what theme its going to change into (Here is a list made by |{–365). The third number is how much time it takes to change theme. (It should always be 1f). Here is what each true/false means:

useTrees
useTowers
useAltGround
usePyramids
usePalmTrees
useSlashers
useBuildings
useFloodLights
useFilmGrainEffect
useAltPalmTrees
useTeslas
useRetroEffects
seGreenMonitorEffects
useColorMonitorEffect

9. After you finish editing click compile

10. Click on file > Save all...

11. Click OK


V: Custom themes (Textures)



1. Open UABE

2. Select file > open

3. Go to your Rolling Sky installation folder > Rolling Sky_Data

4. Select the sharedassets0.assets file and open it

5. Go to view > search by name

6. Search the following “textureX” (X means the level id of the level texture you want to edit and texture means the texture you want to edit. If you are importing a bg, skip to step 10. Here is a list)

7. Select plugins > export to .png

8. Save the texture file somewhere you can remember

9. Edit the texture with your image editor of choice and save it

10. Go back to UABE and select plugin > edit > load

11. Select the edited texture then ok > ok

12. Save the sharedassets0.assets by clicking file > save

Note: UABE wont let you save the sharedassets0.assets in its original location while its open. You have to save it somewhere else and close UABE. After closing UABE you can move the sharedassets0.assets Rolling Sky installation > Rolling Sky_DATA


VI: Custom music (Method made by Pão)



1. Create an Unity Project with nothing just the music.

2. Build the project.

3. Use UABE in "sharedassets0.assets"

4. Select the AudioClips and export as Dump.

5. Rename the Dump to the AudioClip name to be replaced. EX: to change massif rename to "Music1.txt".

6. Change the "sharedassets0.resource" fron the unity project to any name. EX: "CustomMusic.resource"

7. Open the dump file and change the "1 string m_Source = "file.resource"" >> file.resource can be another name, change to the custom filename.

8. Paste the custom file "CustomMusic.resource" in Rolling Sky_Data folder.

9. Using UABE open "globalgamemanagers" and select the music to change. EX: "Music1"

10. Select and import the dump file and save the files.

IX: Changing midgrounds



1. Open dnSpy

2. Select file > open

3. Go to your Rolling Sky installation folder > Rolling Sky_Data > Managed

4. Select the Assembly-CSharp.dll file and open it

5. Select the magnifying glass next to the start button to search

6. Search “MidgroundManager” and double click on the first option

7. Scroll to line 190. In this line you will find the code for all of the themes midground configuration in rs. The code is in order of theme IDs (Theme ID list). It should look something like this:

private static MidgroundManager.BlockData[] blockData = new MidgroundManager.BlockData[]
{
new MidgroundManager.BlockData(5f, 15f, 0f, 12f, 4, new PrefabName[]
{
PrefabName.Block1
}),

(Theme 1 "Massif" midground configuration)

8. Search for the theme you want to change the midground. Right click the code and select "Edit Class (C#)" to edit it. The row of numbers on top is the location and amount of the midground (Hover your mouse over BlockData to know which float does what). To change the model you have to change whats after the PrefabName. (A list of midgrounds has not been created so there will be no list. You will have to find out which model is which yourself) (Some midgrounds use more than one model. You can also add or remove them as you wish)

9. After you finish editing click compile

10. Click on file > Save all...

11. Click OK

XI: Dependent Objects



What are they?:

Dependent objects are objects that vary depending on the current theme. Dependent objects dont have a specific theme ID. They use another already existing object's ID

Example:

Halloween pillar laser

lvl9 pillars


How to edit them:

1. Open dnSpy

2. Select file > open

3. Go to your Rolling Sky installation folder > Rolling Sky_Data > Managed

4. Select the Assembly-CSharp.dll file and open it

5. Select the magnifying glass next to the start button to search

6. Search “ConfigureEnemy” and double click on the first option

7. Search for the dependent object you want to edit. Right click the code and select "Edit Class (C#)" to edit it.
Note: All dependent objects have different code although most include a line of code (LevelDesigner.CurrentWorldThemeNumber == 22 ||) that specifies what theme uses that dependent object.
Note: Remember to remove conflicting dependent objects before adding a new dependent object

8. After you finish editing click compile

9. Click on file > Save all...

10. Click OK
